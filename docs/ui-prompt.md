# 项目目标：
构建一个桌面 Todo 工具的 React 前端 UI。
只会运行在windows平台上。不会运行在移动端。
核心风格： 极简、纯透明、无磨砂感。不要使用毛玻璃效果（No Blur）。追求“文字直接印在壁纸上”的感觉，确保高对比度。

## 样式：

技术规范：
1. 框架：React 18 + TypeScript；
2. 样式：使用 CSS Modules 或 Styled Components，避免全局样式污染；
3. 适配：仅适配 Windows 10/11，分辨率 1920×1080 及以上；
4. 性能：固定模式组件需轻量化，避免频繁重渲染影响桌面流畅度。

样式：
1. 颜色体系：
   - 固定模式：文字默认使用 #FFFFFF（纯白），已完成任务删除线为 #CCCCCC（浅灰），图标背景为 #00000033（黑色半透），确保在任意壁纸下高对比度；
   - 展开模式：背景为纯白（#FFFFFF），可通过透明度滑块调节（范围 0.7-1），文字为 #333333（深灰），已完成任务删除线为 #999999，交互按钮 hover 色为 #EEEEEE；
2. 尺寸规范：
   - 固定模式：字体大小 14px（系统默认小号），图标尺寸 12px×12px（圆角 2px），列表行高 24px，工具栏按钮尺寸 16px×16px；
   - 展开模式：窗口默认宽度 800px、高度 600px（可缩放），左侧菜单栏宽度 180px，字体大小 16px，图标尺寸 14px×14px；
3. 间距规范：
   - 固定模式：列表项左右边距 8px，上下边距 4px，工具栏按钮间距 8px；
   - 展开模式：列表项行高 32px，菜单栏选项上下间距 12px，内容区内边距 16px。

交互细节：
1. 固定模式：
   - 解锁状态：鼠标移入整个组件时，背景变为 #0000001A（极浅黑透），移出后恢复全透；
   - 锁定/解锁按钮：锁定时图标为“锁闭”，解锁时为“开锁”，点击后有 0.2s 视觉反馈（如图标变色）；
   - 任务操作：点击任务文字/图标即可标记完成（删除线），再次点击可取消完成；
2. 展开模式：
   - 滑块交互：透明度滑块范围 0.7-1（0.7 为最低不透明度，确保内容可读），对比度滑块调节文字与背景的对比（范围 1-2，1 为默认）；
   - 窗口行为：最小化后回到固定模式，关闭则退出应用；
   - 菜单栏：点击选项后高亮当前项（背景色 #EEEEEE），默认高亮“首页”。

### Fixed Mode（固定模式 - 桌面）：

定位： 模拟“钉”在桌面背景上。像windows桌面的组件

样式： 背景全透明。无边框、无阴影。

内容： Todo 列表（文字 + 图标）。列表布局，每一行仅有文字+图标，右对齐。图标在右侧，文字在左侧。图标使用圆角方块显示，尺寸比字体尺寸略小。列表最顶部是工具栏，有排序、展开、锁定三个按钮。

锁定/解锁：锁定时除了顶部工具栏的锁定/解锁按钮外，忽略鼠标事件。当解锁时，点击列表项将项目设置为已完成，样式改为删除线。当解锁状态时，鼠标移动到页面上时页面从完全透明改为半透明。



### Expanded Mode（展开模式 - 标准windows窗口）：

定位： windows标准应用窗口。

样式： 带有不透明度、纯白的背景，确保内容易读。与windows窗口一样，顶部左侧有工具栏，顶部右侧有最小化、关闭等按钮

设置： 必须包含“透明度调节”和“对比度调节”的滑块 UI。

视觉规范：
1. 布局：左侧为菜单栏，右侧为主要内容区域。
2. 字体：使用系统默认字体
3. 图标：使用系统默认图标
4. 首页显示Global展示页面，列表方式展示
5. 菜单栏分别是首页、Global、Task
6. task页面展示task列表，列表方式展示，每一行仅有文字+图标，左对齐。图标在左侧，文字在右侧。图标样式与固定模式相同。

页面细化：
1. Global 展示页面：
   - 顶部筛选栏：可筛选“全部/未完成/已完成”任务，支持按“创建时间/截止时间”排序；
   - 列表项：包含“任务名称 + 截止时间（可选） + 状态图标（完成/未完成）”；
2. Global 编辑页面：
   - 编辑项：任务名称（输入框）、截止时间（日期选择器）、延迟原因（文本框，delay task 专用）、日志内容（多行文本框，log 专用）；
   - 操作按钮：保存、取消、删除、标记延迟；
3. Task 查找页面：
   - 日历组件：选择日期后，下方列表仅展示当天的任务，日历默认选中当前日期；
4. Task 关联页面：
   - 布局：左侧展示主任务，右侧分栏展示“关联延迟任务”“关联日志”，无关联时显示“暂无关联内容”；
5. Conclusion 模式：
   - 输入框提示文字：“输入任务完成总结（可选）”，输入后自动保存到任务日志；
6. 时光机页面：
   - 统计维度：今日/本周/本月完成任务数、未完成数、延迟数，以“数字 + 简易图表（如柱状图）”展示；
   - 筛选：可选择时间范围（今日/本周/本月/自定义）。
